Hugo+Github Pages博客搭建

Posted by     "QianCheng" on Saturday, May 1, 2021

Hugo（雨果）是一个Go语言编写的网页创建框架，速度快，主题非常丰富。GitHub Pages是一个免费的静态站点，可以绑定自己的域名，无域名也可以免费使用其支持的username.github.io作为域名。二者结合可以迅速搭建起一个精致而免费的个人博客网站。

本篇将介绍Hugo的安装、主题配置、博文操作、Github Pages部署等基本内容，以及图片、公式、代码高亮等专项配置。不全处可进一步参考Hugo中文文档。

首先需要安装Hugo：直接下载二进制可执行文件、通过包管理工具安装、源代码方式安装等方式均可，可参考安装链接。以Windows系统、直接下载可执行文件的方式为例，我从Hugo releases下载了可执行文件hugo_0.83.0_Windows-64bit.zip，解压、将解压后的路径添加到环境变量即可。

然就可以在任意路径如D:中创建你的博客项目如mySite：

D:\>hugo new site mySite

Hugo就会创建工作路径D:\mySite，且生成如下文件框架：

D:\>cd mySite
D:\mySite>tree/F
D:.
│   config.toml
│
├───archetypes
│       default.md
│
├───content
├───data
├───layouts
├───static
└───themes

接下来我们需要选择一个博客主题：在Hugo主题库中挑选并下载到mySite\themes中，可以下载多个，主题的文档中通常建议用git clone的方式。我个人不建议在工作根目录mySite下初始化git，clone到mySite\theme中的主题又是一个git项目，git嵌套会复杂一些。

我建议的方式是在mySite\theme下初始化git并clone主题，或者在其他路径clone后复制过来，反正弄进去就行。以本博客使用的主题hugo-theme-cleanwhite为例：

D:\mySite>cd themes
D:\mySite\themes>git init
D:\mySite\themes>git clone https://github.com/zhaohuabing/hugo-theme-cleanwhite.git
: 国内githubclone太慢时，可以在gitee(码云)中导入该项目，再从gitee上clone

其实现在就可以初步查看网页效果了：

D:\mySite>hugo server -t hugo-theme-cleanwhite --buildDrafts

-t为指定编译使用的主题，和主题文件夹名一致，运行后便会开启对http://localhost:1313/的监听。在浏览器中进入http://localhost:1313/，可以看到网页的空白模板，什么都没有：

使用主题预置素材和参数（非必须）：为了进一步美化界面，大部分主题其实都提供了包含更多素材的exampleSite文件夹，预置了很多图片素材和参数设置，我们直接基于它来构建和调整我们的博客是更轻松的。将mySite\themes\hugo-theme-cleanwhite\exampleSite中的全部内容覆盖复制到mySite中，鼠标复制粘贴也行：

D:\mySite\themes>xcopy hugo-theme-cleanwhite\exampleSite .. /E /Y

效果一下就丰富了，然后对mySite\config.toml和mySite\static中各类图片素材按照自己的要求进行调整即可。config.toml中的配置项命名很清晰，很方便配置。

接下来介绍对博文的增删改：新增命令如下，会在mySite\content下的post路径（不存在就会新建）中创建内容由test.md控制的博文；编辑该markdown文件即可进行写作；直接删除该文件即可删除博文。

D:\mySite>hugo new post/test.md

Hugo新建的博文markdown文件带有front matter开头，是每篇博客重要的配置信息，可以在其中调整时间、作者等。对本博客的主题cleanwhite而言，其中的categories参数直接控制着页面的顶部栏目名。其生成模板由mySite\themes\hugo-theme-cleanwhite\archetypes中的模板md文件确定。

接下来介绍线上部署：我们使用Github Pages来托管网站，域名固定为username.github.io，username为你Github的用户名，自有域名也可绑定，这里暂不介绍。

首先作部分必要的参数调整：打开配置文件mySite\config.toml

baseurl = "https://username.github.io"  # username为你Github的用户名
theme = "hugo-theme-cleanwhite" # 指定主题

本地编译：

D:\mySite>hugo

该命令会产生一个mySite\public路径，我们需要将这个路径推送到Github远端，他人能看到的页面全来自这个路径中的内容。在Github上新建一个名为username.github.io的公开库，username必须为你自己Github的用户名，然后在mySite\public路径中初始化git，添加提交后推送到远端刚建好的库：

D:\mySite>cd public
D:\mySite\public>git init
D:\mySite\public>git add .
D:\mySite\public>git commit -m "myblog init"
D:\mySite\public>git remote add origin https://github.com/username/username.github.io.git
D:\mySite\public>git push origin master

等待一两分钟后就可以在https://username.github.io 上看到你的页面了。

接下来记录本主题cleanwhite下一些专项问题的解决：图片的路径和大小设置、数学公式的渲染、代码高亮的颜色。或许也可以用在某些其他主题上，仅供参考。

图片的路径：baseUrl默认为mySite\static，markdown中使用\img\...即可定位到mySite\static\img中的图片。

图片的大小设置：markdown的图片插入语法不支持大小和位置调整，只能使用原生html语法<img src="..." width=... />来实现此需求，但原生html会被hugo屏蔽：从 0.60 版本开始，默认的 Markdown 渲染器已经改成了 goldmark，Goldmark 默认不渲染 HTML tags [1]。在mySite\config.toml的[markup]中增加defaultMarkdownHandler = "blackfriday"可以解决这个问题 [2]。

数学公式的渲染：一开始没办法显示行内公式。用mathJax解决，它是一个专门渲染数学公式的JavaScript库。

1.在主题中的themes\hugo-theme-cleanwhite\layouts\partials\head.html中插入mathJax库

<script src="https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.2/MathJax.js?config=TeX-MML-AM_SVG"></script>

2.在markdown开头的front matter配置里增加 markup: mmark（可在archetypes\default.md中修改默认，也可每次在涉及到公式的markdown文件中添加）[3] 。实际测试发现，这和前面提到的blackfriday渲染器并不冲突。

配置后，行间行内公式都用$$...$$表示，如\(f(x)=x^3\)。

代码高亮的颜色：默认的代码块背景色色调太深，并不喜欢，直接在mySite\config.toml中的[markup]- [markup.highlight]中调整style = "xxx"即可，参数可参照风格库。
